Features

Zero-DOM

Aucun layout browser, aucun style recalcul, aucune inflation DOM. Le rendu contourne entierement le moteur CSS.

GPU Instancing

Un seul draw call pour des milliers d'elements. Le scene graph en RAM alimente directement le pipeline WebGPU.

Agnostique

Adapters pour React, Svelte, Angular et Vanilla JS. Props strictes et type-safe pour chaque framework.

Benchmark complet DOM vs Synapse avec rapports JSON/CSV. Stats GPU, CPU, frame budget et percentiles.

Le parcours d'un pixel
1

Le probleme : le DOM

Chaque element passe par HTML → CSSOM → Layout → Paint → Composite. Pour 10k elements le navigateur tombe a 13 FPS.

<div> × 10 000 → layout recalc → 65ms/frame
2

Scene Graph en RAM

Synapse remplace le DOM par un arbre de nodes en memoire. Pas de CSS ni de layout. Juste des positions et couleurs dans un Float32Array.

engine.createBox({ position, size, color })
3

Buffer GPU

Le BufferManager serialise l'arbre en buffers GPU. Chaque node = 24 floats. Un seul writeBuffer() envoie tout au GPU.

device.queue.writeBuffer(instanceBuffer, data)
4

1 Draw Call → 857 FPS

WebGPU dessine 10 000 elements en 1 instanced draw call. Frame budget a 12%, CPU libre. 61x plus rapide que le DOM.

pass.draw(6, 10000, 0, 0) // 1 call, c'est tout
bye DOM

Quickstart
~/SynapseJS
# Clone le repo
$ git clone https://github.com/Klaynight-dev/SynapseJS.git
$ cd SynapseJS

# Installe les dependances
$ bun install

# Lance le dev server
$ bun run dev
  VITE v6.x  ready in 120ms
  -> Local: http://localhost:5173/

# Benchmark Synapse vs DOM
$ bun run bench

# Rapport comparatif complet
$ bun run bench:report

Architecture

  • Synapse-Core — Init WebGPU, scene graph RAM, instancing, raycaster
  • Synapse-Shaders — WGSL primitives UI, SDF texte, effets visuels
  • Synapse-Adapters — React / Svelte / Angular / Vanilla
  • UI Styles — Radius, gradient, shadow — tout via WGSL

Guide pas a pas

1

Prerequis

Un runtime JS (bun, pnpm, npm ou yarn) et un navigateur avec WebGPU (Chrome 113+, Edge 113+, Firefox Nightly).

2

Cloner & installer

Clone le repo et lance l'install. Toutes les deps sont minimales.

3

Dev server

Lance le serveur Vite — la demo vanilla s'ouvre sur localhost:5173.

4

Benchmarks

Lance bench pour Synapse seul ou bench:report pour le comparatif DOM vs Synapse complet.

5

Parametrer les benchmarks

URL params : ?rects=5000&size=10&speed=0.9&gpu=1. Aussi &format=json ou csv pour l'export.
